MULTI-TENANT ARCHITECTURE IMPLEMENTED
================================================================================

USER MODEL ENHANCEMENT:
  restaurant_id (FK to Restaurant, nullable for super_admin)
  is_super_admin (Boolean) - True only for platform owner
  role (String) - super_admin, restaurant_admin, manager, waiter, kitchen
  created_at (DateTime) - Account creation timestamp
  restaurant (relationship) - Points to assigned restaurant

RESTAURANT MODEL (NEW):
  id (PK), name, email, phone, address, city, country, postal_code
  owner_id (FK to User) - restaurant_admin user
  active (Boolean) - Enable/disable restaurant
  created_at, updated_at (DateTime)
  owner (relationship) - FK to user
  staff (relationship) - All users assigned to restaurant

STORE SETTINGS MODEL (NEW):
  id (PK), restaurant_id (FK, unique)
  timezone - Local timezone (e.g., 'Europe/Dublin')
  locale - Language preference (e.g., 'en')
  currency - Transaction currency (e.g., 'EUR')
  tax_region - Tax jurisdiction (e.g., 'EU')
  address_format - Localized address (standard, european, asian)
  business_registration - Business registration number
  vat_number - VAT/GST number for compliance
  payment_terms - Days for payment (0=due on receipt)
  invoice_prefix - Custom invoice numbering (e.g., 'INV')
  created_at, updated_at (DateTime)
